# Como Contribuir

Obrigado por contribuir para este projeto de anotações de Assembly x86_64! Este documento explica o fluxo de contribuição e como manter o repositório organizado.

## Antes de começar
- Leia `README.md` e `COMMIT_GUIDELINES.md`.
- Verifique a estrutura do projeto em `PROJECT_STRUCTURE.md`.
- Certifique-se de estar usando um branch limpo e atualizado.

## Fluxo recomendado
1. Crie um branch a partir de `main`:
   - `feature/<nome>` para novos conteúdos
   - `docs/<nome>` para documentação
   - `fix/<nome>` para correções
2. Faça alterações pequenas e focadas.
3. Use `git add` para incluir apenas os arquivos relevantes.
4. Commite usando o padrão definido em `COMMIT_GUIDELINES.md`.
5. Rebase ou atualize seu branch antes de enviar, se necessário.
6. Faça `git push origin <branch>` e abra um pull request.

## Escrevendo uma boa contribuição
- Mantenha títulos e descrições claros.
- Explique o objetivo da mudança no commit e no PR.
- Inclua exemplos ou referências quando adicionar conteúdo técnico.
- Não misture mudanças não relacionadas no mesmo commit.

## Revisão de PR
- Verifique se o PR está focado em um único tópico.
- Verifique ortografia, formatação e consistência com o restante do repositório.
- Confirme se os links e referências estão corretos.

## Observações
- Use o estilo Markdown consistente com os outros arquivos do repositório.
- Atualize `README.md` se a estrutura do projeto mudar.
- Se encontrar problemas de formatação, corrija-os antes de enviar.
