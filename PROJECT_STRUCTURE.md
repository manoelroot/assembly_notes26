# Estrutura do Projeto

Este documento descreve a organização das pastas e arquivos deste repositório.

## Pastas principais

- `linux_x86_64/`
  - Notas de Assembly x86_64 para Linux.
  - Subpastas incluem: introdução, ferramentas, sintaxe, arquitetura, exemplos, tópicos avançados, imagens e diagramas.

- `docs/`
  - Regras, tipos, convenções e observações sobre Assembly.
  - Arquivos de referência para código seguro e legível.

- `sources/`
  - Recursos de aprendizado, ferramentas e extensões úteis.
  - Ajuda a configurar ambiente e aprofundar os estudos.

- `diagrams/`
  - Diagramas Mermaid para visualização de arquitetura e fluxo de execução.
  - Serve como suporte visual para o conteúdo técnico.

- `images/`
  - Ilustrações SVG que complementam os conceitos abordados no repositório.

## Arquivos de nível raiz

- `README.md` — ponto de entrada do repositório, com visão geral e uso.
- `COMMIT_GUIDELINES.md` — convenções de mensagem de commit e fluxo Git.
- `CONTRIBUTING.md` — orientações para contribuir com o projeto.
- `PROJECT_STRUCTURE.md` — descrição da organização das pastas.
- `.gitignore` — arquivos e padrões ignorados pelo Git.
- `LICENSE` — licença do projeto.

## Como navegar

1. Comece em `README.md` para entender o propósito geral.
2. Veja `PROJECT_STRUCTURE.md` para conhecer a organização do repositório.
3. Consulte `CONTRIBUTING.md` antes de adicionar ou modificar conteúdo.
4. Use `COMMIT_GUIDELINES.md` para manter mensagens de commit consistentes.

## Atualização da estrutura

- Ao adicionar novas pastas, atualize `PROJECT_STRUCTURE.md`.
- Ao mudar regras de commit ou contribuição, atualize `COMMIT_GUIDELINES.md` e `CONTRIBUTING.md`.
- Mantenha `README.md` sincronizado com o layout e os principais pontos do repositório.
